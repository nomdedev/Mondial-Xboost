Cómo funciona paso a paso

1

Recolectamos datos históricos

Descargamos CSVs de football-data.co.uk con resultados de 21 ligas (Europa, América, Asia) desde 2012. También datos de mundiales de Wikipedia. Son 42,315 partidos reales con goles, tiros, córners, tarjetas y cuotas de apuestas.

2

Calculamos features (variables predictivas)

Para cada partido, calculamos 23 features usando solo información anterior al partido. Ejemplo: "en los últimos 10 partidos de Argentina, ¿cuántos goles promedió?". Incluye Elo ratings (como eloratings.net), forma reciente, y enfrentamientos directos.

3

Dividimos en 3 grupos (nunca mezclamos)

Train (antes de 2023): el modelo aprende de estos. Validation (2023): ajustamos hiperparámetros. Test (2024+): evaluación final, el modelo NUNCA vio estos datos. Es como rendir un examen con preguntas que nunca viste.

4

Entrenamos XGBoost con Optuna

XGBoost es el motor canónico del proyecto. Optuna prueba cientos de combinaciones de hiperparámetros para encontrar la configuración que mejor generaliza a partidos futuros.

5

Optimizamos con Optuna (hyperparameter tuning)

En vez de adivinar la configuración, Optuna prueba automáticamente combinaciones de parámetros (profundidad del árbol, learning rate, regularización...) y aprende cuáles funcionan mejor. Como un científico que prueba 1000 experimentos.

6

Loop Engineering: repetimos 1000 veces

Cada batch de 100 iteraciones analiza qué funcionó, ajusta la búsqueda, y repite. Después de 1000 iteraciones, tenemos el mejor modelo posible. Cada ciclo mejora el anterior.

Feature Engineering

Las 23 variables que usamos para predecir cada partido

¿Qué es una feature?

Es un dato numérico que el modelo usa para aprender. Ejemplo: "promedio de goles de Argentina en los últimos 10 partidos" = 1.8. El modelo aprende que equipos con mayor promedio de goles tienden a ganar más.

Ejemplo: Si Argentina promedia 1.8 goles y Brasil 1.2, la diferencia (0.6) es una feature predictiva.

Elo Ratings

Sistema de rating dinámico (como eloratings.net). Se actualiza después de cada partido.

E_A = 1 / (1 + 10^((R_B - R_A + 100) / 400))

R_new = R_old + K × (Resultado - Esperado)

K varía: Mundial=60, Continental=50, Liga=30, Amistoso=20. Home advantage: +100 puntos.

Forma Reciente

Rolling windows de 5 y 10 partidos. Solo usa partidos ANTERIORES (shift 1).

• Promedio de puntos (normalizado)
• Goles anotados / recibidos
• Tasa victoria / empate / derrota
• Partidos jugados

✓ Anti-leakage: shift(1) garantiza que no se ve el partido actual

Head-to-Head

Historial directo entre los dos equipos que se enfrentan.

• Último resultado entre ellos
• Promedio de goles en H2H
• Diferencia de victorias
• Años desde último enfrentamiento

✓ Anti-leakage: solo incluye partidos anteriores al actual

Algoritmos de Machine Learning

Los 5 algoritmos que compiten para predecir cada partido

¿Qué hace un algoritmo de ML?

Le mostramos 30,000 partidos con sus features (Elo, forma, H2H) y el resultado real (ganó local, empate, ganó visitante). El algoritmo aprende patrones: "cuando el Elo diff es > 100 Y la forma reciente es buena, el local gana el 65% de las veces". Luego predice partidos nuevos.

XGBoost

Principal

Construye cientos de "árboles de decisión" simples. Cada árbol corrige los errores del anterior. Es como tener 500 expertos donde cada uno opina y el siguiente aprende de los errores de los anteriores.

Predicción = Árbol_1 + Árbol_2 + ... + Árbol_500 (cada uno pesado por su contribución)

Hyperparameter Tuning

Cómo encontramos la mejor configuración de cada algoritmo

¿Qué son los hiperparámetros?

Son las "perillas" que controlan cómo aprende el algoritmo. Ejemplo: ¿Cuántos árboles construir? ¿Qué tan profundos? ¿Qué tan rápido aprender? No hay una respuesta universal — depende de los datos. Por eso Optuna prueba automáticamente miles de combinaciones.

Analogía: Cocinar una receta

Imaginá que cocinás un plato. Los ingredientes son las features (datos). Los hiperparámetros son las cantidades: ¿cuánta sal? ¿A qué temperatura? ¿Cuánto tiempo? Optuna prueba 1000 combinaciones de cantidades y encuentra la que hace el plato más rico.

Cómo funciona Optuna

1

Sampling

Optuna prueba una combinación aleatoria de hiperparámetros.

2

Entrena y evalúa

Entrena el modelo con esa configuración y mide accuracy en validation set.

3

Aprende (TPE Sampler)

Optuna usa Tree-structured Parzen Estimator: aprende qué regiones del espacio funcionan mejor y muestrea más de esas regiones.

4

Repite

20 trials por modelo × 5 modelos = 100 configuraciones evaluadas por batch. Después de 10 batches = 1000 configuraciones totales.

Métricas

Accuracy

¿Cuántas veces acertó?

correctas / total

Random: 33% | Actual: 57.6% | Target: 85%

Log Loss

Penaliza confianza errónea

-Σ(y_true × log(y_pred)) / N

Si predice 90% y falla → muy mal. Ideal: < 1.0

Brier Score

Error cuadrático de probabilidades

Σ(pred - real)² / N

Mide calibración. Ideal: ≈ 0.20

Loop Engineering

1000 iteraciones de mejora continua: 10 batches × 100 iteraciones

¿Por qué 1000 iteraciones?

Un solo entrenamiento prueba UNA configuración. Con 1000 iteraciones, probamos 1000 configuraciones diferentes. Cada batch de 100 analiza qué funcionó y enfoca la búsqueda en las regiones prometedoras.

¿Qué hace cada batch?

1

Optuna Tuning (20 trials × 5 modelos = 100)

Optuna prueba 20 configuraciones por algoritmo. Usa TPE (Tree-structured Parzen Estimator) para aprender de los resultados anteriores y enfocar la búsqueda.

2

Evaluación en Validation Set

Cada configuración se evalúa en datos de 2023 (que el modelo nunca vio). Se registra accuracy, log loss y Brier score.

3

Walk-Forward Validation

El mejor modelo de cada algoritmo se evalúa con 3 folds temporales. Esto verifica que no overfitteó a un período específico.

4

Análisis y guardado

Se guarda el mejor modelo, los mejores hiperparámetros, y el historial completo. El siguiente batch usa estos resultados como punto de partida.

Tracking de progreso

Cada batch guarda JSON con todos los resultados. Podemos ver la mejora a lo largo del tiempo:

batch_1_best: 57.6% → batch_2_best: 59.2% → ... → batch_10_best: 65%+?
Cada batch debería mejorar el anterior (o al menos mantenerlo)

Calibración de probabilidades

Temperatura Scaling

Ajusta la confianza del modelo. Si dice "80% gana local" pero solo acierta el 60% de las veces, la temperatura corrige eso.

P_calibrada = softmax(logits / T)

Home Advantage

El equipo local tiene ventaja natural. En nuestro Elo ya sumamos +100 puntos al local. Varía por liga (~10-15%).

dr = (elo_home - elo_away) + 100

Data Leakage Audit

Verificación de que el modelo no "hace trampa" viendo datos del futuro

¿Qué es data leakage?

Es cuando el modelo accidentalmente "ve" información del futuro. Ejemplo: si para predecir "Argentina gana" usamos el resultado del partido como feature, el modelo siempre acierta — pero no sirve para predecir partidos nuevos.
